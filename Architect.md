# 🏥 MedPlatform

Единое медицинское веб-приложение: запись к врачу, электронная медкарта, лабораторные исследования.

> Учебный командный проект · 7 человек · 3–4 недели

## Модули

### Appointment — Запись к врачу
- Расписание врача и доступные слоты
- Создание / отмена записи
- Статус приёма

### EHR — Электронная медкарта
- Карточка пациента
- История визитов и заметки врача
- Назначения

### Lab — Лабораторные исследования
- Создание направления на анализ
- Статусы: `назначен → взят → готов`
- Просмотр результата

### Telemed — Телемедицина _(опционально)_
- Создание онлайн-консультации
- Обмен сообщениями
- Фиксация итога

---

## Роли пользователей

| Роль | Что может |
|------|-----------|
| `patient` | Просматривать свои записи, ЭМК, анализы |
| `doctor` | Вести расписание, создавать записи ЭМК, назначать анализы |
| `lab_technician` | Обновлять статусы и вносить результаты анализов |
| `admin` | Полный доступ, управление пользователями |

---

## Сквозной сценарий

```
1.  Пациент записывается к врачу
2.  Врач проводит приём → создаёт запись в ЭМК
3.  Врач назначает анализ
4.  Лаборант берёт материал → обновляет статус
5.  Лаборант вносит результат
6.  Пациент просматривает результат
7.  Врач открывает телемед-консультацию
8.  Обмен сообщениями
9.  Врач фиксирует итог консультации
```

## API — общие правила

- Все эндпоинты за авторизацией (JWT)
- Единый формат ошибок по всему бэкенду
- Контракты фиксируются в `docs/api/` до начала реализации
- Изменения контрактов — только через согласование команды

---

## Нефункциональные требования

- **RBAC** — каждый эндпоинт защищён ролевой проверкой
- **Audit Log** — все значимые действия логируются
- **Валидация** — входные данные проверяются на бэкенде
- **Идемпотентность** — повторный запрос не создаёт дубликат
- **Тесты** — smoke и интеграционные сценарии для критичных путей
- **CI** — пайплайн запускается на каждый PR
