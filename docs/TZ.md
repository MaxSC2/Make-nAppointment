# Практическое задание: MedPlatform (командная разработка)

## О проекте

Единое медицинское веб-приложение MedPlatform, объединяющее 4 функциональных направления:

- Запись к врачу
- Электронная медицинская карта (ЭМК)
- Лабораторные исследования
- ~~Телемедицина (Не делать)~~

Проект выполняется в одном контуре: общий backend, общий frontend, общая модель пользователей и ролей, единые API-стандарты и сквозные бизнес-сценарии.

**Срок выполнения:** 3–4 недели.

---

## Функциональный минимум (MVP)

### 1. Запись к врачу
- расписание врача
- доступные слоты
- создание / отмена записи
- статус приёма

### 2. ЭМК
- карточка пациента
- история визитов
- заметка врача
- назначения

### 3. Лаборатория
- создание направления на анализ
- статусы: назначен → взят → готов
- просмотр результата анализа

## Нефункциональные требования

- Ролевая модель доступа (пациент, врач, лаборант, администратор)
- Журналирование ключевых действий (Audit Log)
- Валидация входных данных и доменных ограничений
- Единый формат ошибок backend + корректная обработка на frontend
- Наличие smoke/integration-тестов для критичных сценариев

---

## Обязательные инженерные практики и паттерны

### Backend
- Controller → Service → Repository
- DTO
- RBAC
- Audit
- Идемпотентность критичных операций

### Frontend
- feature-based модули
- shared components
- route guards
- typed API layer
- централизованная обработка loading/error

### Командная разработка
- contract-first по API
- изменения контрактов только через согласование
