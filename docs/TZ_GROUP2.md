# Практическое задание: SmartQ — Система управления очередью
## Вторая группа (4 человека)

**Цель:** Разработать MVP интеллектуальной системы электронной очереди, которая управляет потоком людей, кабинетов и специалистов на основе событий, правил и аналитики.

## Основные модули

### 1. Queue Engine / Ядро очереди
Создание талона, присвоение типа услуги, приоритет, расчёт времени ожидания, выбор кабинета/специалиста, смена статуса, история событий.

**Статусы талона:** created → waiting → called → in_service → completed / cancelled / no_show / redirected

### 2. Event-driven логика
Каждое действие сохраняется как событие: ticket_created, patient_arrived, ticket_called, service_started, service_completed, ticket_cancelled, queue_overloaded и т.д.

### 3. Smart Routing
Система выбирает кабинет с учётом: тип услуги, загрузка кабинетов, среднее время обслуживания, приоритет пациента, перегрузка.

### 4. ETA / Прогноз времени ожидания
Формула: ETA = количество пациентов перед ним × среднее время обслуживания по типу услуги.
Типы услуг: consultation (15 мин), payment (5 мин), xray (10 мин), analysis (7 мин), other (12 мин).

### 5. Rule-based Assistant
Рекомендации: перегрузка кабинетов, превышение времени ожидания, узкие места.

### 6. Dashboard руководителя
Показатели: талонов за день, среднее время ожидания/обслуживания, нагрузка по кабинетам, отмены, no-show, предупреждения.

### 7. Realtime табло
Экран для телевизора: номер талона, кабинет, статус. Обновление через WebSocket или polling 3-5 сек.

### 8. Роли
- admin — создаёт талоны, видит очередь, перенаправляет
- specialist — очередь своего кабинета, вызывает, завершает
- manager — dashboard и аналитика

### 9. API
POST/GET /tickets, POST /tickets/{id}/call, POST /tickets/{id}/complete и т.д.

### 10. AI-driven (перспектива)
Анализ загрузки кабинетов, времени обслуживания, no-show, скорости врачей, пробок по времени, поведения пациентов — динамическое управление потоком.
