# ТЗ — РИС: Радиологическая Информационная Система

> Статус: черновик для обсуждения с куратором
> Команда: группа АнтисМед, 6 человек
> Срок: 3–4 недели

---

## Что это за система

РИС — отдельное веб-приложение которое связывает несколько участников
радиологического процесса: врач направляет пациента на снимок,
рентгенолог делает снимок, результат возвращается врачу.

Интеграция с MedPlatform: данные о пациентах и направлениях
могут приходить из MedPlatform через API.

---

## Участники системы (роли)

| Роль | Что делает в системе |
|---|---|
| **Врач** | Создаёт направление на исследование, смотрит результат |
| **Рентгенолог** | Видит очередь, меняет статусы, загружает снимок/результат |
| **Пациент** | (опционально) Видит свои направления и результаты |

## Основной сценарий (сквозной)

Врач создаёт направление → В очереди у рентгенолога → В процессе → Готово → Врач видит результат

## MVP

### 1. Направления
- Создание: пациент, тип исследования, приоритет, комментарий
- Фильтры по статусу, дате, типу
- Статусы: Назначено → В очереди → В процессе → Готово → Отменено

### 2. Очередь
- Список пациентов на сегодня
- Сортировка по приоритету и времени
- Кнопки смены статуса
- Счётчики: всего / ожидают / в процессе / готово

### 3. Исследования
- Карточка: пациент, тип, дата, статус, исполнитель
- Загрузка файла + текстовое заключение
- Просмотр результата

### 4. Мониторинг
- Статистика за день
- Среднее время ожидания
- Разбивка по типам

### 5. DICOM (опционально)
- Загрузка .dcm
- Просмотр через DWV / OHIF
