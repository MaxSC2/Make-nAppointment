## Стек технологий проекта

- **Frontend:** React 18 + Vite + TypeScript
- **Стилизация:** Tailwind CSS
- **Роутинг:** React Router v6

Для пациента — страница лаборатории (вкладки, поиск, статусные бейджи, скелетоны загрузки) и EMR (электронная медкарта: боковое меню с историей визитов и назначениями). ### 30.05.2026 — Методология и организация работы

**Что делал:**
Обсуждали с командой как будем работать — договорились об общем подходе.

Договорились работать итерациями:
1. Сначала фронтенд на моках (захардкоженные данные или localStorage)
2. Параллельно бэкенд поднимает API
3. После готовности API — заменяем моки на реальные запросы

Это позволяет фронту и бэку работать независимо и не блокировать друг друга.

По Git договорились: каждый работает в своей ветке (`feature/queue-ui`, `feature/orders-ui` и т.д.), изменения вливаются в `main` через Pull Request после проверки. Общие файлы (типы, конфиги) — только через согласование, чтобы не было конфликтов.

Инструменты которые использую ежедневно:
- **VS Code** — основной редактор
- **Git + GitHub** — версионирование, ветки, PR
- **Docker Desktop** — запуск PostgreSQL и Orthanc локально
- **Swagger UI** (`localhost:8000/docs`) — проверка API без Postman
- **React DevTools** — отладка компонентов
- **AI-ассистент (Claude Code)** — генерация кода, архитектурные решения, ускорение разработки. Использую как пару-программиста: объясняю задачу, проверяю результат, правлю под наш стек и требования.

### 05.06.2026 (продолжение) — Системный анализ отклонений, проектирование двухуровневого хранения снимков (Tiering) и автоматизация запуска системы

Сегодня провёл глубокий системный аудит проекта совместно с AI-кодером. На основе замечаний куратора и анализа файла `GAP_ANALYSIS.md` зафиксировал ключевое отклонение: текущая система работает как «дневной регистратор очереди» (~40% отклонения от цели), в то время как целевой продукт для врачей должен быть полноценной медицинской информационной системой (МИС/EHR), накапливающей данные пациентов долгосрочно.

1. **Проектирование Hot/Cold Storage Tiering:** Описал концепцию деления снимков на «горячие» (активные в Orthanc на быстрых SSD) и «холодные» (старые снимки >30 дней, сжимаемые без потерь через JPEG-LS и выгружаемые на NAS/S3). Разработал алгоритмы автоматической фоновой заморозки (freezing) и быстрого восстановления (rehydration/recall) по запросу врача.
2. **Интеграция ИИ-ассистента для DICOM:** Спроектировал API-контракт `POST /api/v1/studies/{uid}/ai-analyze` для интеграции локальных ONNX-моделей сегментации патологий и VLM-моделей для генерации черновика протокола, что сократит время написания заключений на 70%.
3. **Оптимизация и автоматизация запуска:** Для исключения конфликтов портов и упрощения развёртывания всей сложной системы (из 6 сервисов) написал единый корневой скрипт-оркестратор `start-all-services.bat`. Он автоматически очищает порты от зависших процессов, поднимает PostgreSQL в Docker, накатывает миграции Alembic, инициализирует СУБД и параллельно запускает все компоненты (Vite React, FastAPI RIS, FastAPI Queue, Orthanc PACS, NestJS DICOMweb, статический сервер вьювера), а при закрытии — чисто завершает процессы.
