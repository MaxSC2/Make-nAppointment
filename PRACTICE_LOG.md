В панели три действия: «Назначить анализ» (6 шаблонов — общий анализ крови, биохимия и т.д.) и «Завершить приём» (жалобы, диагноз, назначения). ### 30.05.2026 — Методология и организация работы

**Что делал:**
Обсуждали с командой как будем работать — договорились об общем подходе.

Договорились работать итерациями:
1. Сначала фронтенд на моках (захардкоженные данные или localStorage)

3. После готовности API — заменяем моки на реальные запросы

Это позволяет фронту и бэку работать независимо и не блокировать друг друга.

По Git договорились: каждый работает в своей ветке (`feature/queue-ui`, 
`feature/orders-ui` и т.д.), изменения вливаются в `main` через Pull Request 
после проверки. Общие файлы (типы, конфиги) — только через согласование, 
чтобы не было конфликтов.

Инструменты которые использую ежедневно:
- **VS Code** — основной редактор
- **Git + GitHub** — версионирование, ветки, PR
- **Docker Desktop** — запуск PostgreSQL и Orthanc локально
- **Swagger UI** (`localhost:8000/docs`) — проверка API без Postman
- **React DevTools** — отладка компонентов
- **AI-ассистент (Claude Code)** — генерация кода, архитектурные решения,
  ускорение разработки. Использую как пару-программиста: объясняю задачу,
  проверяю результат, правлю под наш стек и требования.
