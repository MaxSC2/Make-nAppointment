# Правила оформления PRACTICE_LOG.md

## 1. Общие требования к отчёту (из .docx)

| Параметр | Значение |
|---|---|
| Формат листа | A4 |
| Шрифт | Times New Roman, 14 pt |
| Выравнивание | По ширине |
| Межстрочный интервал | 1.25 (полуторный) |
| Поле левое | 30 мм |
| Поле правое | 20 мм |
| Поле верхнее | 10 мм |
| Поле нижнее | 25 мм |
| Нумерация страниц | Арабские цифры, на титульном номере нет |
| Абзацный отступ | 1.25 см |
| Проверка орфографии | Word + Grammarly |

## 2. Структура отчёта (из .docx)

1. **Титульный лист** — без номера страницы
2. **Введение** — актуальность, цели, задачи
3. **Основная часть** — разделы (1, 2, 3...) + подразделы (1.1, 1.2...)
4. **Заключение** — выводы
5. **Список литературы** — книги, статьи, нормативные документы, электронные ресурсы
6. **Приложения** — A, B, C...

### Оформление заголовков

| Уровень | Пример | Форматирование |
|---|---|---|
| Раздел | `1. Название раздела` | Полужирный, прописные |
| Подраздел | `1.1 Название подраздела` | Полужирный |
| Пункт | `1.1.1 Название пункта` | Полужирный |

- Расстояние между заголовком и текстом — **не менее 5 строк**
- Заголовки и подразделы нумеруются арабскими цифрами
- Точка в конце заголовка **не ставится**

## 3. Требования к самому PRACTICE_LOG.md

### Формат даты
`DD.MM.YYYY`

### Структура каждой записи

```
## DD.MM.YYYY — Краткий заголовок

### Что сделал:
[подробное описание]

### Какие файлы создал/изменил:
[список файлов]

### Результат:
[итог: тесты, TSC, что заработало]

### Что это дало для отчёта:
- **П.N — название:** [обоснование]
```

### Обязательные поля для каждой записи
- ✅ Заголовок с датой
- ✅ «Что сделал» (описание работы)
- ✅ «Какие файлы создал/изменил»
- ✅ «Что это дало для отчёта» (привязка к пунктам силлабуса)

### Необязательные, но желательные
- «Какие были сложности» (проблемы и их решения)
- «Результат» (цифры: тесты, ошибки TSC, скриншоты)

## 5. Оформление рисунков и таблиц

### Рисунки
- Нумерация сквозная или по разделам: `Рисунок 1`, `Рисунок 2.1`
- Подпись **снизу** рисунка
- Ссылка в тексте: `(Рисунок 1)` или `на рисунке 1`

### Таблицы
- Нумерация сквозная или по разделам: `Таблица 1`, `Таблица 1.1`
- Подпись **сверху** таблицы
- Ссылка в тексте: `(Таблица 1)` или `в таблице 1`

### Формулы
- На отдельной строке, выравнивание по центру
- Номер в круглых скобках справа: `(1)`, `(2.1)`

## 6. Оформление списка литературы

**Книга:**
```
Автор И.О. Название. — Город: Изд-во, год. — С.
```

**Статья из журнала:**
```
Автор И.О. Название // Журнал. — год. — Т. N, № N. — С. NN-NN.
```

**Электронный ресурс:**
```
Заголовок. URL. Дата обращения.
```

**Патент:**
```
Пат. 2187888 РФ, МПК...
```

**Автореферат / диссертация:**
```
Автор И.О. Название: дис. ... канд. наук. — Город, год. — С.
```

## 7. Оформление приложений

- Каждое приложение — на отдельной странице
- Обозначаются A, B, C... (кроме I, O, Ё, Й)
- Допускается обозначение русскими буквами (А, Б, В...)
- Ссылки в тексте: `(Приложение А)`

---

## 8. Проверки перед коммитом PRACTICE_LOG.md

- [ ] Дата актуальна (не в будущем, не пропущена)
- [ ] Все обязательные поля заполнены
- [ ] Есть привязка к П.1–П.10
- [ ] Шрифт/поля не применимы к .md (конвертировать в docx перед сдачей)
- [ ] Нет орфографических ошибок (проверить Word/Grammarly)
- [ ] Коммит с сообщением `docs: PRACTICE_LOG.md — DD.MM.YYYY — кратко`

```
[2-3 абзаца с деталями]

- ✅ TypeScript 0 ошибок
- Что заработало

### Что это дало для отчёта:
- **П.N — название:** обоснование
- **П.N — название:** обоснование
```
