## 1. Цель проекта

Автоматизация работы рентгенологического отделения:
- Электронная очередь пациентов
- Назначение и выполнение исследований (КТ, МРТ, Рентген, УЗИ)
- Подписание протоколов исследований
- Мониторинг загрузки кабинетов и врачей
- Поддержка 3 языков: русский, английский, казахский

## 6. Функциональность

### 6.1 Электронная очередь (`/`)
- Отображение очереди по кабинетам
- Фильтрация по кабинету
- Вызов следующего пациента (Call)
- Завершение приёма (Complete)
- Автообновление каждые 10 секунд

### 6.2 Регистрация пациента (`/register`)
- Создание талона с ФИО, полисом, кабинетом, приоритетом
- Автоматическая очередь

### 6.3 Кабинет врача (`/doctor`)
- Выбор кабинета
- Просмотр активных талонов
- Вызов / завершение
- Скелетон загрузки

### 6.4 Заказы RIS (`/orders`)
- Список заказов с фильтрацией по статусу
- Просмотр деталей заказа
- Связь с очередью SmartQ

### 6.5 Пациенты (`/patients`)
- Поиск по ФИО или полису
- Карточка пациента с историей исследований
- Назначение нового исследования

### 6.6 Исследования (`/studies`)
- Просмотр снимков из PACS (Orthanc)
- Связывание снимков с заказами
- DICOM Viewer (DWV)

### 6.7 Протоколы (`/protocol/:orderId`)
- Создание/редактирование протокола исследования
- Черновик и подпись
- После подписи изменения невозможны

### 6.8 Мониторинг (`/monitoring`)
- KPI: заказы сегодня, завершено, в работе, в очереди
- Подписанные протоколы, среднее время обработки
- Загрузка кабинетов
- Активность врачей
- Автообновление каждые 30 секунд

### 6.9 Локализация
- 3 языка: RU, EN, ҚАЗ
- Переключатель в хедере
- Все страницы переведены

## 7. API Endpoints (REST)

### 7.1 Auth
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/login` | Вход (username + password → JWT) |
| GET | `/api/auth/me` | Текущий пользователь |
| POST | `/api/auth/refresh` | Обновление токена |

### 7.2 Очередь
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/queue/tickets` | Список талонов (фильтр по кабинету) |
| POST | `/api/queue/tickets` | Создать талон |
| POST | `/api/queue/tickets/{id}/call` | Вызвать пациента |
| POST | `/api/queue/tickets/{id}/complete` | Завершить приём |
| GET | `/api/queue/cabinets` | Список кабинетов |

### 7.3 Заказы / Пациенты / Исследования
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/orders` | Список заказов |
| POST | `/api/orders` | Создать заказ |
| GET | `/api/orders/{id}` | Детали заказа |
| GET | `/api/patients` | Список пациентов |
| GET | `/api/patients/{id}` | Карточка пациента |
| GET | `/api/patients/{id}/studies` | Исследования пациента |
| GET | `/api/studies` | Список исследований |
| POST | `/api/studies/{id}/link` | Связать с заказом |
| GET | `/api/protocol/{orderId}` | Протокол |
| PUT | `/api/protocol/{orderId}` | Сохранить черновик |
| POST | `/api/protocol/{orderId}/sign` | Подписать протокол |

### 7.4 Статистика
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/monitoring/summary` | Сводка KPI |
| GET | `/api/monitoring/by-modality` | По модальностям |
| GET | `/api/monitoring/by-cabinet` | По кабинетам |
| GET | `/api/monitoring/physicians` | Активность врачей |

## 8. База данных (ключевые таблицы)

### `users`
- `id` (UUID, PK), `username`, `password_hash`, `full_name`, `email`
- `is_active`, `is_superuser`, `last_login_at`, `created_at`

### `user_roles`
- `user_id` (FK → users), `role_code` (FK → roles)

### `roles`
- `code` (PK: `admin`/`doctor`/`registrar`/`technician`), `name`, `description`

### `orders`
- `id` (UUID, PK), `patient_id`, `modality`, `priority`, `status`
- `referring_doctor`, `note`, `created_at`, `completed_at`
- Связь с SmartQ: `source_ticket_id`, `source_ticket_number`, `source_system`

### `protocols`
- `id` (UUID, PK), `order_id` (FK), `findings`, `impression`
- `signed_by` (FK → users), `signed_at`, `created_at`, `updated_at`

### `patients`
- `id` (UUID, PK), `full_name`, `policy_number`, `birth_date`, `phone`
